Você sabe o que é um CGI?

CGI (Common Gateway Interface) é um padrão que permite que servidores web interajam com programas externos para produzir páginas web dinâmicas. Ele é amplamente usado para processar dados de formulários e criar conteúdo dinâmico.

Considerações importantes sobre CGI:

  • Como você não chamará o CGI diretamente, use o caminho completo como PATH_INFO.
  • Lembre-se de que, para solicitações "chunked", seu servidor precisa desfragmentá-las. O CGI espera EOF (Final de Arquivo) como fim do corpo da mensagem.
  • O mesmo vale para a saída do CGI. Se nenhum "content_length" for retornado do CGI, o EOF marcará o fim dos dados retornados.
  • Seu programa deve chamar o CGI com o arquivo solicitado como primeiro argumento.
  • O CGI deve ser executado no diretório correto para acesso a arquivos com caminho relativo.
  • Seu servidor deve funcionar com um CGI (como php-CGI, Python e assim por diante).

Entender a CGI é crucial para desenvolver aplicações web dinâmicas e garantir que os servidores web funcionem corretamente com programas externos.
